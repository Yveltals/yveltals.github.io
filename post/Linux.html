Linux命令

网络与进程

  • netstat [选项] 查看网络状态，-i网卡信息，-anp | grep port查看占用端口进程
  • ifconfig 显示或设置网络设备，配置IP、MAC地址、启动网卡
  • iftop 显示进程级别的网络IO
  • fdisk 创建和维护分区表
  • tcpdump 抓包
  • route 查看路由
  • pgrep dummy 查看进程号，-l显示进程名
  • pkill dummy Kill进程名，-f全匹配command line
  • ps aux | grep -w 'hello' | grep -v grep | awk '{print $2}' | xargs kill -9 Kill进程名的另一种方式
  • pstree 以树状图的形式现实系统中的进程

文件查找与操作

  • sed -i "s/查找字段/替换字段/g" src.txt
  • grep {xxx} src.txt
  • less 浏览文件
  • head -n log.txt 查看前 n 行
  • tail -f log.txt 查看尾行（实时刷新）
  • which 查找某一命令所在的路径,格式为 which 命令
  • whereis 命令与其类似,但是 whereis 还能展示命令帮助文档所在的路径
  • find 格式为 find {path} -name {xxx}
  • locate 功能与 find -name 相同,find 是遍历了搜索路径下的每个文件,而 locate 是搜索/var/lib/locatedb 里的一个数据库,相当于散列查找。在使用 locate 命令之前,可先使用 updatedb 命令手动更新数据库。
  • headlink -f . 查看当前路径
  • mkdir -p 依次创建目录
  • wget下载,-c断点续传,-b后台下载（wget-log）
  • zcvf src_dir/ dst.tar.gz 压缩
  • zxvf xxx.tar.gz -C dst_dir/ > /dev/null 2>&1 解压缩
  • mount挂载
    • 硬盘:mount 设备 挂载点
    • 镜像:mount -o rw -t iso9660 设备 挂载点,rw读写,ro只读

通配符

通配符含义实例
*匹配任意字符串file_*.txt，匹配file_qseawd.txt等
?匹配一个字符file_?.txt，匹配file_1.txt等
[abc]匹配abc其中一个字符file_[ab].txt，匹配 file_a.txt 和 file_b.txt
[a-z]匹配范围中一个字符file_[a-z].txt，匹配 file_a.txt 至 file_z.txt
[^abc]匹配之外的字符file_[^ab].txt，排除 file_a.txt, file_b.txt

C/C++编译

g++编译参数

  • -g 编译带调试信息的可执行文件

  • -o 指定输出文件名

  • -O2 优化源代码

  • -Wall 打印警告信息，-w 关闭警告信息

  • -std=c++11 设置编译标准

  • -D 定义宏

  • -l 指定库文件 -L指定库文件路径

  • -I 指定头文件搜索目录

Cmake

指定特定版本gcc

服务器上有多个版本gcc时，cmake可能会使用老版本的gcc/g++
通过 which gcc 和 gcc/g++ --version 确定正确版本路径后，在cmake命令时加入对应gcc和g++的位置：

重要指令

  • add_library - 生成库文件

  • add_compile_options - 添加编译参数

  • add_executable - 生成可执行文件

  • add_subdirectory - 向工程添加存放源文件的子目录，其中需有一个CMakeLists.txt

  • include_directories - 引入头文件目录

  • link_directories - 引入库目录，如工程编译时可能用到第三方库的 lib 文件

  • link_libraries - 引入库文件到当前工程（全路径）

  • target_link_libraries - 引入库文件到子工程，即分配，故需指定子工程

  • target_include_directories - 引入头文件目录到子工程，同5
