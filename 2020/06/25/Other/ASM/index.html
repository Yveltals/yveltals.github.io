div byte ptr ds:[0] //8位 (al)=((ax)/(ds)*16+0)的商 (ah)=(ax)/((ds)*16+0)的余数

 (dx)=(ax)*((ds)*16+(bx)+8)结果的高16位

转移指令

可以修改IP或同时修改CS，IP的系统指令：

  • 转移行为：
    1. 只修改IP，称为段内转移，如jmp ax
    2. 同时修改CS和IP，称为段间转移，如jmp 1000:0
  • 修改范围（段内转移）：
    1. 短转移：修改IP范围-128~127
    2. 近转移：修改IP范围-32768~32767
  • 转移指令分类：
    1. 无条件转移：jmp
    2. 条件转移：jcxz
    3. 循环指令：loop
    4. 过程
    5. 中断

offset指令

offset start : 取得偏移地址

jmp指令

jmp short 标号：位移字类型8位，段内短转移

jmp near ptr 标号：位移字类型16位，段内近转移

jmp far ptr 标号：段间远转移，机器码中指明目的地址的CS和IP值

jcxz指令

jcxz 标号：条件转移指令，短转移，如果(cx)=0则跳转

loop指令

loop 标号：循环指令，短转移，如果(cx)!=0那么跳转到标号处执行

ret/retf/iret指令

用栈中的数据修改CS和IP的值，实现近、远转移

  • ret 指令相当于 pop ip

    (IP)=((SS)*16+(SP))
(SP)=(SP)+2

  • retf 指令相当于 pop ip, pop cs

    (IP)=((SS)*16+(SP))
(SP)=(SP)+2
(CS)=((SS)*16+(SP))
(SP)=(SP)+2

  • iret 指令相当于 pop ip, pop cs, popf

call指令

将当前的IP或CS和IP入栈，转移，且位移字类型16位

call不能实现短转移，但它实现转移的原理和jmp相同。

  • call 标号相当于 push ip, jmp near ptr 标号
  • call far ptr 标号 相当于 push cs, push ip, jmp far ptr 标号

标志寄存器

CPU内部特殊寄存器，用来存储指令执行结果，简称flag寄存器，按位起作用

ZF 零标志位，结果是否为0

PF 奇偶标志位，结果二进制位中1的个数是否偶

SF 符号标志位，结果是否为负；得知有符号数结果正负，对无符号数无意义

CF 进位标志位，记录无符号数运算是否向更高位进/借位

OF 溢出标志位，记录有符号数运算是否发生溢出

TF 调试标志位，为1时处理器每次只执行一条指令，即单步执行

IF 中断允许标志位，为1时能响应外部中断，反之则屏蔽

DF 方向标志位，为0则si、di递增，为1递减；cld命令置为0，std置为1

adc   带进位加法，adc ax, bx 即(ax) = (ax)+(bx)+CF
//计算001EF000H + 00201000H，结果放在ax(高16位)和bx(低16位)中
mov ax, 001EH
mov bx, 0F000H
add bx, 1000H
adc ax, 0020H   
//高位: ax = ax + 0020H +1  (CF保存了进位1)
sbb   带借位减法，sbb ax,bx 即(ax)=(ax)-(bx)-CF

cmp 比较指令，cmp ax,bx即(ax)-(bx)，结果正负靠SF(正负)和OF(溢出)共同判断

端口

什么是端⼝

各种存储器都和CPU的地址线、数据线、控制线相连。CPU在操控它们的时候，把它们都当作内存来对待，把它们总的看做⼀个若⼲存储单元组成的逻辑存储器，这个逻辑存储器我们称其为内存地址空间。

在PC机系统中，和CPU通过总线相连的芯⽚除各种存储器外，还有以下3种芯⽚。

  • 各种接⼝卡（⽐如，⽹卡、显卡）上的接⼝芯⽚，它们控制接⼝卡进⾏⼯作；
  • 主板上的接⼝芯⽚，CPU通过它们对部分外设进⾏访问；
  • 其他芯⽚，⽤来存储相关的系统信息，或进⾏相关的输⼊输出处理。

在这些芯⽚中，都有⼀组可以由CPU读写的寄存器。这些寄存器都和CPU的总线相连，CPU进⾏读或写的时候都通过控制线向它们所在的芯⽚发出端⼝读写命令。可⻅，从CPU的⻆度，将这些寄存器都当作端⼝，对它们进⾏统⼀编址，从⽽建⽴了⼀个统⼀的端⼝地址空间。

CPU可以直接读写以下3个地⽅的数据：CPU内部的寄存器、内存单元、端⼝


端口的读写

在访问端⼝的时候，CPU通过端⼝地址来定位端⼝，因为端⼝所在的芯⽚和CPU通过总线相连，所以，端⼝地址和内存地址⼀样，通过地址总线来传送。在PC系统中，CPU最多可以定位64KB个不同的端⼝。则端⼝地址的范围为0～65535

端⼝的读写指令只有两条： in 和 out ，分别⽤于从端⼝读取数据和往端⼝写⼊数据

对比CPU执⾏内存访问指令和端⼝访问指令时候，总线上的信息：

  1. 访问内存

    mov ax,ds:[8]                 ;假设执⾏前(ds)=0
    • CPU通过地址线将地址信息8发出
    • CPU通过控制线发出内存读写命令，选中存储器芯⽚，并通知它，将要从中读取数据
    • 存储器将8号单元中的数据通过数据线送⼊CPU

  2. 访问端⼝

    in al,60h                   ;从60h号端⼝读⼊⼀个字节
    • CPU通过地址线将地址信息60h发出
    • CPU通过控制线发出端⼝读命令，选中端⼝所在的芯⽚，并通知它，将要从中读取数据
    • 端⼝所在的芯⽚将60h端⼝中的数据通过数据线送⼊CPU

注意：

在 in 和 out 指令中，只能使⽤ ax 或 al 来存放、发送到端⼝中的数据。
访问8位端⼝时⽤ al ，访问16位端⼝时⽤ ax 。


CMOS RAM芯⽚的读写

PC机中，有⼀个CMOS RAM芯⽚，⼀般简称为CMOS。此芯⽚的特征如下:

包含⼀个实时时钟和⼀个有128个存储单元的RAM存储器（早期的计算机为64个字节）

该芯⽚靠电池供电。所以，关机后其内部的实时时钟仍可正常⼯作，RAM中的信息不丢失

128个字节的RAM中，内部实时时钟占⽤0～0dh单元来保存时间信息，其余⼤部分单元⽤于保存系统配置信息，供系统启动时BIOS程序读取。BIOS也提供了相关的程序，使我们可以在开机的时候配置CMOS RAM中的系统信息

该芯⽚内部有两个端⼝，端⼝地址为70h和71h。CPU通过这两个端⼝来读写CMOS RAM

70h为地址端⼝，存放要访问的CMOS RAM单元的地址；71h为数据端⼝，存放从选定的CMOS RAM单元中读取的数据，或要写⼊到其中的数据。可⻅，CPU对CMOS RAM的读写分两步进⾏。⽐如，读CMOS RAM的2号单元：将2送⼊端⼝70h；从端⼝71h读出2号单元的内容

利用中断和栈实现loop

原题：在屏幕中间显示80个！

参数：(cx)=循环次数，(bx)=位移

assume cs:code
code segment
start:
    mov ax,cs
    mov ds,ax
    mov si,offset lp
    mov ax,0
    mov es,ax
    mov di,200h

    mov cx,offset lpend-offset lp ; 安装中断例程
    cld
    rep movsb

    mov word ptr es:[7ch*4],200h
    mov word ptr es:[7ch*4+2],0 ; 设置中断向量表

    mov ax,0b800h
    mov es,ax
    mov di,160*12
    mov bx,offset s-offset se ;设置回退位移
    mov cx,80            ;设置loop次数
s:mov byte ptr es:[di],'!' ; 完成第一次显示
    add di,2
    int 7ch ;测试int 7ch
se:nop

    mov ax, 4c00h
    int 21h

lp: ;中断例程
    dec cx
    jcxz f
    push bp   ;修改bp前先保存
    mov bp,sp  ;SP指向栈顶，即存入的IP
    add [bp+2],bx  ;指针回退位移....为什么+2？？？？
f: pop bp
    iret
    lpend:nop

code ends
end start

3.子程序要先将所用到的寄存器存到栈里

4.使用'xxx'的方式指明数据是以字符形式给出的，编译器会自动将它们转化为ASCII码
